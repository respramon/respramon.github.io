Selamat datang dalam pelajaran pertama belajar binary exploitation! Hari ini kita akan belajar tentang konsep buffer overflow.

Apa itu Buffer Overflow?

Buffer adalah area memori tempat program menyimpan data, misalnya teks yang dimasukkan pengguna. Buffer overflow terjadi saat data yang masuk melebihi kapasitas buffer tersebut sehingga merusak data di memori di sekitar buffer, dan ini bisa dimanfaatkan untuk mengubah perilaku program.

Contoh Program C yang Rentan Buffer Overflow

Perhatikan program C berikut ini yang sangat sederhana:

#include <stdio.h>
#include <string.h>

int main() {
    char buffer[8];

    printf("Masukkan input: ");
    gets(buffer);

    printf("Kamu mengetik: %s\n", buffer);
    return 0;
}

Penjelasan:

  • char buffer[8]; artinya kita membuat buffer hanya cukup untuk 8 karakter.
  • gets(buffer);, jangan menggunakan gets di program nyata! Fungsi ini membaca input user tanpa batas maksimum yang menyebabkan buffer overflow jika input lebih dari kapasitas buffer.

Eksperimen Buffer Overflow

Coba jalankan program ini, lalu masukkan lebih dari 8 karakter, misalnya: AAAAAAAAAAAAAAAAAAAA

Apa yang terjadi? Bisa jadi program crash (error), atau menampilkan data yang tidak diharapkan, karena memori ter-overwrite.

Catatan Penting: Program ini hanya contoh untuk belajar. Pada kode nyata, gets() dilarang karena riskan keamanan. Kita menggunakan ini untuk memahami konsep dasar.

Langkah Selanjutnya

  1. Compile program C ini di Linux/macOS dengan: gcc -o overflow lesson1.c -fno-stack-protector -z execstack
  2. Jalankan program dengan ./overflow
  3. Coba input panjang > 8 karakter dan lihat efeknya.
Tip: Saya akan membimbing kamu membuat program ini langkah demi langkah, dan menjelaskan kenapa terjadi buffer overflow serta bagaimana eksploitasi dilakukan setelahnya.

Jika kamu sudah mengerti, kita bisa lanjut ke pelajaran berikutnya untuk belajar bagaimana memanfaatkan buffer overflow untuk menjalankan kode berbahaya.
